人工智能、机器学习和数据科学已经成为大大小小众多企业的必备话题。随着模式识别、计算机视觉和自然语言处理的进步，可以优化的众多流程将推动下一代业务的发展。由于这个原因，许多组织都吵着要把他们的人工智能战略整合在一起。而数据科学家和软件工程师经常通过项目所需的任务来计算项目的难度；企业领导人必须权衡他们选择的许多人事和财务权衡。财务和会计中一个常用的流程是**制造或购买决策。**这是一个考虑内部开发或外包的所有当前和未来成本的过程。本文将解释这些决定的双方的考虑，因为它们与人工智能和数据科学工作有关。

# **在人工智能中做出购买决定**

> **制造或购买决策:**制造或购买决策是在内部制造产品或从外部供应商处购买产品之间做出选择的行为。[1]

制造/购买决策的定义是不言自明的，在人工智能的特定用例中，选项如下:

**Make:** 对个人收集的数据从头开始实施机器学习模型——这包括构建神经网络架构、编写定制算法、利用其他 ML 方法和构建 API 来服务结果。

**购买:**为使用现有 API 付费，编写最少的代码来预处理现有服务并与之交互，以实现公司目标——这包括但不限于微软 Azure 或亚马逊 AWS AI 服务，用于 CV、NLP 和许多其他任务。

# **每条路径的限制**

**制作**

大公司和小公司的许多 IT 领导通常都精通当前的顶级技术。他们的好奇心和对技术社区的参与通常是他们走到这一步的原因。出于这个原因，他们经常会选择超出公司整体目标范围的尖端项目。在这些情况下，他们可能会推动人工智能工作，让高度复杂的项目从零开始构建可能已经存在的人工智能技术。虽然这对团队来说是令人兴奋的，但结果往往并不完美。

我见过的一个例子是一家小型医疗服务公司，他们希望从头开始构建自己的转录(语音到文本)人工智能系统，以便与更大的应用程序进行交互。任何有经验的数据科学家都会认识到，构建一个能够在复杂的医疗操作中工作的 NLP 系统需要大量的数据和时间，其性能优于当前的语音到文本系统。无论他们被建议如何尝试利用像[AWS transcript](https://aws.amazon.com/transcribe/)这样的服务，IT 领导们继续要求在内部构建它。进一步细分，开发的最终成本可能包括 5+的员工，在计算其他成本之前，每月的工资消耗率为 20，000+美元。另一方面显示，现有的转录服务可以以低于 1 美元的价格获得许可！

不难看出，这种“制造”决策会如何受益于“购买”决策。相反，如果任务涉及复杂和独特的数据类型，那么花费就会合理得多。同样，如果企业希望将这种转录服务作为核心产品销售，他们也会从构建自己的知识产权中受益。当我为企业的人工智能战略提供建议时，我总是会提出这些观点。认识到像微软、谷歌或亚马逊这样的企业在开发他们的服务上投入的时间和人员是很重要的。一个核心关注点不是技术的企业不太可能取得同样的结果，更不用说降低成本了。

**购买**

如上所述，购买决策可以提供高质量的人工智能服务，其成本和时间只是内部开发的一小部分。对于希望在内部业务流程中利用人工智能的企业来说，这是一个很好的选择。事实是，大多数希望利用人工智能的公司本质上并不是技术公司，因此利用技术公司开发的模型比立即建立一个高技能的 IT 部门更有效。

然而，购买决定也有缺点。数据不兼容和缺乏数据都是这些服务的障碍。数据不兼容是指复杂的数据结构或复杂的预期输出。许多这样的服务期望完美组织的数据产生一个通用的输出。当需要高于语音转录或图像分类水平的任务时，它们通常变得无用。在这些情况下，数据科学家将需要做足够的预处理工作，以证明构建定制模型来适应任务的潜力。考虑系统在整个生命周期中的适应性也很重要。如果计划预计会超出能力(即边缘/物联网要求)，则考虑内部构建以消除这种依赖性是很重要的。最后，数据法规经常会妨碍与第三方 API 的交互，这可能是导致倾向于“做出”决策的另一个要考虑的因素。

# **几个“购买”选项**

这里是一些可用服务的快速列表，这些服务可以很好地集成，以有效地满足许多数据科学的需求。

*   微软 Azure 人工智能(Microsoft Azure AI):Azure 基于云的人工智能套件为计算机视觉、自然语言处理和聚类领域的任务提供 API。大多数服务都是基于 REST API 的。也有工具来构建聊天机器人和许多其他常见的人工智能用例。
*   [AWS 机器学习](https://aws.amazon.com/machine-learning/):另一家公认的云提供商，拥有众多 API。除上述服务之外，还有文档文本理解、预测、HIPAA 认证转录和欺诈检测。
*   [H20](https://www.h2o.ai) : H20 提供数据工作室“让你的公司成为 AI 公司”。它们提供多个软件套件，采用跨行业数据类型并完成常见任务。

这个列表可能会占用一整篇文章来解释每种方法提供的机会和细微差别。

总之，可以说将 AI 实施到任何生产系统中都需要数据科学家和机器学习工程师团队的深厚专业知识。虽然 it 部门建立自己的人工智能系统很诱人，但考虑已经存在的选项通常更有效。这有助于避免大量的成本和麻烦。另一方面，如果一个组织有时间、资源和人员来试验一个完全不同的系统，这可能有利于在这个垂直领域创造下一个大事件。我确信，10 年后将会有大量的案例研究详细说明这些决策中的对错。像汽车这样的老行业可能会发现，通过与更多以技术为导向的合作伙伴合作来填补这些空白，他们在 autonomy 内部建设的努力本可以加快。花时间考虑你的组织是否最适合制造或购买你的下一个人工智能应用程序，可能是这一代任何商业转型中最重要的决定。
