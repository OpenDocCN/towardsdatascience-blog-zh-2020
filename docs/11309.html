介绍

Twitch 是一个直播视频流平台，允许用户观看和广播直播或预先录制的广播公司视频游戏游戏的视频。该平台由亚马逊的子公司 Twitch Interactive 拥有和运营。Justin.tv 成立于 2011 年，是“流一切平台”(stream everything platform)的后代，主要专注于流媒体直播视频游戏，包括播放 Twitch 自己主办的电子竞技比赛。除了这些功能，Twitch 还播放音乐和创意内容等，这些内容可以在网站上直播或通过“视频点播”观看。

截至 2019 年第四季度，Twitch 的平均月播放量超过 360 万，略低于 2020 年 3 月的 380 万。Twitch 平均有超过 56，000 个并发广播公司和 144 万个并发观众，Twitch 的 web APIs 每秒处理超过 5 万个请求，相当于 2019 年观看了超过 23 亿小时。

Twitch 的数据科学团队利用从不同的日常数据流中生成的大量数据，进行了广泛的分析，以帮助制定产品决策。这一壮举是通过增强的数据管道实现的，“该管道每天收集数据、清理数据并将超过 10 亿个事件加载到他们的数据仓库中。”

Twitch 的数据科学职位

Twitch 有一个“科学团队”，由与数据科学相关的头衔和角色组成。它由三大支柱支持，即“数据科学研究、用户体验研究和数据治理”。数据科学正好位于这三个科学组织的中间，并且在许多情况下与其他团队合作。

Twitch 的数据科学家角色在很大程度上受到他们所合作的团队的影响，因此，角色和职能可能包括从以产品为中心的分析到机器学习和深度学习算法。目前，有两种主要类型的数据科学家:一种是“以产品战略为导向的数据科学家”，他们从数据分析中提供业务影响洞察；另一种是“数据产品数据科学家”，他们构建特定的算法和技术，根据数据生成新产品。

所需技能

Twitch 只雇用在数据科学相关项目方面至少有 3 年(高级数据科学家职位为 5 年以上)行业经验的合格数据科学家。

注意:申请的处理和评估是基于与团队工作角色相关的特定行业经验。

其他相关资格包括:

  • 计算机科学、数学、统计学或相关领域的高级学位(硕士或博士)。
  • 对大规模数据应用监督和非监督算法的经验。
  • 扎实的数据科学基础知识背景:R 或 Python 中的数据操作、SQL 和统计学(假设检验、回归等)。).
  • 应用于大规模数据的机器学习技术的高级知识。
  • 具有构建数据管道、数据仓库、维度建模、构建聚合和优化数据工作流程(从数据准备到分析再到部署)的经验。
  • AWS 中的声音背景。
  • 有使用数据库管理系统软件的经验，如 MySQL，PostgreSQL 等。
  • 有在产品开发周期中设计和评估 A/B 实验影响的经验。
  • 在 Tableau 或类似软件中创建仪表板的高度舒适性。
  • 具有数据分析和数据沟通的经验，包括实验、数据可视化和定义业务的 KPI 战略。
  • 有使用 R、Python、C/C++、Go 或 Java 等编程语言的经验。
  • 在注册渠道、参与度指标和留存分析方面的产品分析经验。
  • 在具有复杂数据集的业务环境中，精通 SQL 和 ETL/ELT。

Twitch 有哪些数据科学团队？

虽然 Twitch 有一个专门的“科学团队”，由数据科学家、数据分析师和数据工程师组成，但数据科学家通常嵌入在其他团队中，有时会与其他部门合作。作为一家拥有超过 20 个团队的数据科学家的大公司，在个人层面上，Twitch 的角色与特定的团队有着内在的联系。

根据团队的需求，Twitch 的数据科学家角色可能包括:

  • 广告产品:利用一系列先进的数据分析工具和方法来解决广泛的挑战性问题，包括计量经济学建模和拍卖动态、定价和细分，以及为观众和广告商实现价值最大化。
  • 核心产品:建立和跟踪关键绩效指标，设计实验，评估 A/B 测试和实施数据工具，以制定战略和评估/改进产品计划。
  • 联盟:定义和跟踪 KPI，支持战略计划，评估新的业务线，并帮助确定 Twitch 交易的绩效评估方式。
  • 财务规划和分析:监督数据工具、设计仪表板/报告构建和指标审查，以指导财务决策并提供业务影响洞察。
  • 营销:领先的事件营销方案分析与优化。与走向市场角色进行跨职能合作，包括增长、产品、内容活动和品牌营销团队。
  • 移动:通过定义 KPI、设计实验、评估 A/B 测试和支持战略计划，在移动团队中制定战略和评估产品计划。
  • 核心科学:通过数据分析和机器学习模型，在整个 Twitch 中推动基于证据的决策。与数据科学、分析、工程和经济学科的专家合作，有效开发大规模的可靠且可重现的分析。

面试过程

Twitch 的数据科学家面试流程从招聘人员的电话面试开始，然后是 45 分钟的技术筛选，然后是现场面试。

初始屏幕

Twitch 最初的数据科学家屏幕面试是与招聘经理进行 30 到 60 分钟的电话聊天，讨论团队、Twitch 作为一个社区、你对数据的想法、你的技术背景以及你过去的相关项目和经验如何与团队中的工作角色保持一致。

样题:

  • 如果你有所有可能的数据和无限的回旋余地，你会对 Twitch 的网站做什么第一个改变？
  • 你做过的一件让你自豪的非学术性的事情是什么？
  • 你最大的弱点是什么？

技术屏幕

Twitch 的数据科学家技术屏幕与大多数科技公司非常相似。这次采访包括一个数据科学家在 coderpad 上进行的一个小时的现场筛选，提问的问题通常是基于 SQL 的。在这次面试中还有一个行为和背景经历的因素。

现场面试

现场面试是 Twitch 数据科学家面试流程中的最后一个面试阶段。本次访谈包括六次一对一(或会议)，围绕行为、实验、SQL 和编码与产品经理、数据科学家、技术产品经理和分析师的访谈。

每次面试大约 45 分钟，这些面试中的问题倾向于实验、A/B 测试、商业智能和深度分析。还有以产品为重点的和行为方面的问题，问题围绕商业分析经验、过去的工作经验以及你对 Twitch 文化的了解。

注意事项和提示

Twitch 的数据科学家面试是数据科学概念的组合，标准化以评估申请人应用统计和分析概念来理解和预测用户行为的能力，并根据分析回答业务问题。它有助于提高你的统计和概率、时间序列分析、实验设计、A/B 测试和预测建模概念的知识。了解 Twitch 使用的指标也很有帮助，尤其是那些与产品和功能相关的指标。

Twitch 提供了一个生态系统，通过鼓励员工弄脏自己的手，找到自己喜欢的东西，让员工茁壮成长，成为最好的自己。很多重点放在通过导师项目建立高绩效团队，事实上，导师的能力和愿望是 Twitch 在申请人身上寻找的东西。

Twitch 数据科学面试问题:

  • 使用 SQL 创建直方图/bin 数据。
  • 在特定的 A/B 测试中，您会跟踪哪些指标？
  • 您会针对什么指标进行优化？你会如何挑选 A/B 测试的“赢家”？
  • 你如何创建垃圾箱？
